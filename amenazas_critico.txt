# Escenario DISCRIMINANTE - disenado para comparar estrategias de planificacion.
# Formato:  ZONA;tiempoRestanteMs;instanteLlegadaMs
#
# Idea: las amenazas a zonas CRITICAS (hospital, central) llegan con bastante
# tiempo (1900 ms), mientras que las de zonas de BAJA criticidad llegan URGENTES
# (1000 ms). Con N=2 y recarga=1000 el sistema esta saturado: solo se pueden
# interceptar 4 de 10, asi que la estrategia decide CUALES se salvan.
#
#   - SRTN puro (solo urgencia) atiende primero las urgentes (las de baja
#     criticidad) y deja que IMPACTEN 2 HOSPITALES.
#   - La estrategia por criticidad (Prioridades / MLQ) protege a los hospitales
#     y centrales: misma cantidad interceptada, pero MENOR criticidad impactada.
#
# Se ve claramente al correr:
#   java Simulador amenazas_critico.txt 2 1000 1   (criticidad: salva hospitales)
#   java Simulador amenazas_critico.txt 2 1000 3   (SRTN: sacrifica hospitales)
#   java Simulador batch amenazas_critico.txt
